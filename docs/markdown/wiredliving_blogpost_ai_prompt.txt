FORMATTING BEST PRACTICES
═══════════════════════════════════════════════════════════════════════════════

TEXT FORMATTING:
• **Bold** for emphasis on key points and terms
• *Italic* for subtle emphasis or quoted phrases
• ~~Strikethrough~~ for showing "before/after" changes
• `Inline code` for technical terms, file names, commands

LISTS:
• Use `-` for unordered lists (consistent throughout)
• Use `1.` for ordered/numbered lists
• Use `- [ ]` and `- [x]` for task lists
• Add blank lines before and after lists

LINKS & IMAGES:
• [Descriptive link text](url) - never use "click here"
• ![Descriptive alt text](image-url) - for accessibility
• Include at least 1 internal link: [Related post](/blog/slug)
• Include 1-2 external links if they add value
• All images need meaningful alt text

SPACING:
• One blank line between paragraphs
• One blank line before/after headings
• One blank line before/after lists, tables, code blocks
• Two blank lines before major sections (H2)
